    突破校园网限制：用 WireGuard 搭建你的专属 VPN！

    OCTOPUS
    小技巧！
    2025/09/02 21:37:12

    文字数：---

    阅读数：

    Abstract:

    你是否也遇到过这样的窘境：在宿舍里，和同学的设备明明连接着同一个 Wi-Fi，却仿佛身处两个世界，无法互相访问？无论是想和小伙伴联机打一把游戏，或是方便地共享文件，甚至是远程访问自己搭建在宿舍的树莓派，都会因为设备间无法 ping 通而宣告失败。这背后的“罪魁祸首”，便是校园网为了网络安全和管理而普遍启用的 AP 隔离（AP Isolation）技术。

    别担心！本文将手把手带你利用一台云服务器和现代、轻量、高效的 WireGuard 工具，轻松搭建一个属于你自己的虚拟局域网（VPN），彻底突破校园网的限制，让你名下的所有设备都能自由地“握手”和通信。

    准备工作：

    1. 一台云服务器：主流的云服务商均可（如阿里云、腾讯云、AWS、Vultr等），许多厂商还为学生提供专属优惠。为了获得最佳体验，建议选择地理位置靠近你学校的机房，以确保更低的延迟。
      • 操作系统：推荐选择 Ubuntu 20.04 LTS 或 22.04 LTS，这两个版本对 WireGuard 的支持更为原生和稳定。
      • 重要提醒：请务必确保云服务器的安全组（或防火墙） 规则已经放行了 WireGuard 计划使用的端口。本文后续教程将使用默认的 UDP 51820 端口。这是新手最容易踩的坑，没有之一！
    2. 客户端设备：你需要在所有希望加入这个虚拟局域网的设备上安装 WireGuard 客户端。它几乎支持所有平台（Windows, macOS, Linux, Android, iOS）。

    第二部分：配置客户端设备

    步骤 1: 为每个客户端生成自己的密钥对

    在Windows/macOS/Android/iOS的WireGuard官方客户端中，当你点击“添加一个空隧道”时，软件会自动为你生成一对全新的公私钥。 这种方法最简单安全，因为私钥从未离开过你的设备。

    你也可以在服务器上为每个客户端生成密钥对（操作与生成服务器密钥类似），但这需要你将生成的私钥安全地分发到对应的客户端设备上。

    现在，我们来规划一下虚拟局域网的IP地址。假设你有两台设备需要互联：一台PC和一台笔记本电脑（Laptop）。

    • PC:
      • 公钥: PC_PUBLIC_KEY
      • 私钥: PC_PRIVATE_KEY (由PC上的客户端生成或从服务器分发)
      • 分配的VPN IP: 10.0.0.2/32
    • Laptop:
      • 公钥: LAPTOP_PUBLIC_KEY
      • 私钥: LAPTOP_PRIVATE_KEY
      • 分配的VPN IP: 10.0.0.3/32

    常见问题与排查 (Troubleshooting)

    1. 客户端无法连接，或连接后没有数据传输（transfer 为0）

      • 99% 的问题都出在这里：云服务器的防火墙/安全组！ 立刻、马上、现在就去你的云服务商控制台检查，确保你已经为服务器的公网IP添加入站规则，允许 UDP 协议的 51820 端口的流量通过。

    2. 客户端之间无法 ping 通，但都能 ping 通服务器（10.0.0.1）

      • 检查IP转发：在服务器上运行 cat /proc/sys/net/ipv4/ip_forward，确保结果是 1。如果不是，请重新执行上面的第五步。
      • 检查iptables规则：确认 /etc/wireguard/wg0.conf 文件中的 PostUp 和 PostDown 命令没有写错，特别是公网网卡名称（eth0 或 ens3 等）是否正确。
      • 检查客户端配置：确保所有客户端的 [Interface] 部分的 Address 子网掩码都是 /24，而 [Peer] 部分服务器的 AllowedIPs 是 10.0.0.0/24。

    3. 连接不稳定，时断时续

      • 这很可能是NAT设备超时导致的。请确保你的客户端配置文件中包含了 PersistentKeepalive = 25 这一行，它能有效维持连接的活性。

    4. 连接速度慢

      • 这主要取决于你的云服务器的带宽和地理位置。选择按量付费模式的服务器通常能获得更高的瞬时带宽。如果速度不理想，可以尝试更换到离你更近的机房或换一家云服务商。
